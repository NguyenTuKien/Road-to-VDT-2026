Kubernetes MCQ (Multiple Choice Questions)

This Kubernetes MCQ collection covers important topics like deployment, networking, storage, scaling, security, and monitoring, helping beginners and experts prepare for interviews, certifications, and strengthen their skills in managing containers and cloud-based applications.

1. What is Kubernetes?
a) A programming language for developing applications
b) A storage management system
c) A tool for managing virtual machines
d) A container orchestration platform
View Answer

Answer: d
Explanation: Kubernetes is an open-source platform designed for managing, scaling, and automating the deployment of containerized applications. It abstracts away the underlying infrastructure and provides tools for automating deployments, scaling, and ensuring the availability of applications.

2. What is a Pod in Kubernetes?
a) A network configuration
b) A group of one or more containers
c) A storage unit
d) A single container
View Answer

Answer: b
Explanation: A Pod is the smallest and most basic deployable unit in Kubernetes. It can contain one or more containers that share the same network and storage resources. Pods are used to run applications and manage containers together within a single unit.

3. Which component of Kubernetes manages the desired state of the cluster?
a) kube-apiserver
b) kube-controller-manager
c) etcd
d) kube-scheduler
View Answer

Answer: b
Explanation: The kube-controller-manager is responsible for managing the desired state of the cluster, including ensuring that the number of Pods matches the desired count and that updates are applied correctly.

4. What is a Kubernetes Cluster?
a) A group of machines running Kubernetes components
b) A single container
c) A network switch
d) An application repository
View Answer

Answer: a
Explanation: A Kubernetes Cluster is a collection of machines (nodes) running Kubernetes components, responsible for running containerized applications.

5. What is a Service in Kubernetes?
a) A tool for managing containers
b) A container registry
c) A way to expose Pods to the network
d) A way to manage storage volumes
View Answer

Answer: c
Explanation: A Service is an abstraction that allows Pods to be accessed over the network. It provides a stable endpoint (IP address and DNS name) for communication, enabling load balancing and routing traffic to the correct Pods.

6. What is a Kubernetes Deployment?
a) A set of instructions for application deployment
b) A configuration object for resource limits
c) A Pod replication controller
d) A group of containers running together
View Answer

Answer: a
Explanation: A Deployment in Kubernetes defines how to deploy and scale a set of Pods, managing updates and rollouts.

7. What is a Node in a Kubernetes Cluster?
a) A worker machine where containers run
b) A database server
c) A networking component
d) A storage device
View Answer

Answer: a
Explanation: A Node is a worker machine in a Kubernetes cluster where containers are deployed and run. It hosts one or more containers and communicates with the control plane.
advertisement

8. What is a Namespace in Kubernetes?
a) A method for controlling access to resources
b) A tool for managing logs
c) A way to group related objects in a cluster
d) A type of network protocol
View Answer

Answer: c
Explanation: A Namespace in Kubernetes is a way to partition and organize resources within a cluster. It allows multiple users or teams to work in the same cluster without interfering with each other, and also enables the management of resources at a finer granularity.

9. What does a ConfigMap in Kubernetes do?
a) Manages network traffic
b) Stores large datasets
c) Manages application secrets
d) Decouples configuration from the application code
View Answer

Answer: d
Explanation: A ConfigMap stores non-sensitive configuration data, decoupling it from the application code for easier updates.

10. Which API server endpoint exposes Kubernetes functionality to developers?
a) kube-apiserver
b) controller-manager
c) kubelet
d) scheduler
View Answer

Answer: a
Explanation: The kube-apiserver acts as the central interface to communicate with Kubernetes clusters, exposing Kubernetes functionality.

11. What is a ReplicaSet in Kubernetes?
a) A method to ensure a specified number of Pod copies
b) A deployment strategy
c) A network policy manager
d) A storage component
View Answer

Answer: a
Explanation: A ReplicaSet ensures that a specified number of identical Pods are running at all times. It is commonly used to provide high availability for applications by maintaining the desired number of Pod replicas.

12. What does the kubectl delete command do?
a) Lists resources
b) Deletes a specified resource
c) Applies a configuration change
d) Displays resource details
View Answer

Answer: b
Explanation: The “kubectl delete” command is used to delete a specified resource, such as a Pod, Deployment, or Service, from the Kubernetes cluster.

13. What is a common reason for container startup failures in Kubernetes?
a) Corrupted images
b) Network misconfiguration
c) Insufficient resources
d) All of the mentioned
View Answer

Answer: d
Explanation: Container startup failures can be due to issues with corrupted images, networking, or insufficient node resources.

14. What is a Kubernetes Secret?
a) A secure storage for sensitive information
b) A load balancer
c) A container replica
d) A tool for deployment
View Answer

Answer: a
Explanation: Secrets store confidential information like passwords and API tokens in Kubernetes, ensuring data security.

15. What is the use of kubectl exec in troubleshooting?
a) Enable auto-scaling
b) Access a running container’s shell to debug issues
c) Configure storage manually
d) Restart nodes
View Answer

Answer: b
Explanation: kubectl exec allows you to access a container’s shell, which is useful for debugging and checking logs interactively.

16. What is the role of the Kubernetes API Server?
a) Acting as the main control point for communication in the cluster
b) Scheduling and deploying containers
c) Managing storage volumes
d) Load balancing within pods
View Answer

Answer: a
Explanation: The API Server is the central control plane component in Kubernetes. It serves as the communication hub where all interactions with the cluster occur, including requests from kubectl commands, controllers, and other components.

17. What is the function of a Kubernetes Ingress?
a) Store logs for pods
b) Expose services to external users and provide HTTP routing
c) Control memory usage across nodes
d) Deploy pods across multiple nodes
View Answer

Answer: b
Explanation: The Ingress resource in Kubernetes allows you to expose multiple services through a single IP address. It handles external access, provides HTTP routing, and often includes features like SSL termination and URL-based routing.

18. What happens when you use the kubectl scale command?
a) Automatically upgrades the cluster nodes
b) Increases or decreases the number of replicas for a deployment
c) Deletes unused services
d) Updates Kubernetes API components
View Answer

Answer: b
Explanation: The kubectl scale command adjusts the number of replicas for a deployment, statefulset, or daemonset. This allows you to scale applications up or down based on traffic or resource availability.

19. What is the purpose of a Kubernetes Network Policy?
a) To set storage limits for containers
b) To allocate CPU resources
c) To control network traffic to and from Pods
d) To monitor cluster performance
View Answer

Answer: c
Explanation: Kubernetes Network Policies define how Pods communicate with each other and external services. These policies enable fine-grained control over network traffic within a cluster.

20. What is a Persistent Volume (PV) in Kubernetes?
a) Temporary storage for Pods
b) A volume specific to a single container
c) A cluster-wide storage resource
d) A network policy configuration
View Answer

Answer: c
Explanation: A Persistent Volume (PV) is a cluster-wide resource that provides durable storage independent of the Pod lifecycle. Pods can claim this storage using Persistent Volume Claims (PVCs).

21. What is the purpose of Kubernetes Labels?
a) Store environment variables
b) Encrypt container data
c) Define application architecture
d) Group and organize pods and other resources
View Answer

Answer: d
Explanation: Labels in Kubernetes are key-value pairs that group and organize resources. They help with querying and filtering, enabling you to select and operate on subsets of pods, services, and other Kubernetes objects.

22. How do you check the status of pods in your Kubernetes cluster?
a) kubectl monitor pod
b) kubectl create pod
c) kubectl start pod
d) kubectl get pods
View Answer

Answer: d
Explanation: The kubectl get pods command provides a list of all pods running in the cluster along with their status, IP addresses, and resource usage information, which helps in monitoring and troubleshooting.

23. What is a DaemonSet in Kubernetes?
a) Stores cluster logs
b) Provides database backups
c) Deploys a pod on all nodes in a cluster
d) Manages service discovery
View Answer

Answer: c
Explanation: A DaemonSet ensures that a pod is running on every node in the cluster. It is typically used for cluster-wide services such as logging, monitoring, or network tools.

24. What does the kubectl describe command do?
a) Provides detailed information about a Kubernetes resource
b) Creates a new node
c) Deploys applications automatically
d) Deletes a pod
View Answer

Answer: a
Explanation: The kubectl describe command displays detailed information about Kubernetes resources like pods, services, deployments, and nodes, including their configurations and status.

25. What is a Blue-Green deployment in Kubernetes?
a) A strategy that keeps two environments: current (blue) and new (green)
b) Deploying to all Pods simultaneously
c) Using multiple replicas of the same Pod
d) Scaling resources up and down automatically
View Answer

Answer: a
Explanation: A Blue-Green deployment strategy involves maintaining two environments: the current one (blue) and the new one (green). Traffic is switched to the green environment once it is verified, ensuring zero downtime.

26. What does a Rolling Update do in Kubernetes?
a) Scales down the cluster
b) Updates Pods incrementally
c) Deletes all Pods and deploys new ones
d) Deploys only one replica
View Answer

Answer: b
Explanation: Rolling Update ensures that Pods are updated incrementally, replacing old Pods with new ones gradually to ensure minimal downtime and uninterrupted service.

27. What is the purpose of the CNI (Container Network Interface) plugin in Kubernetes?
a) Deploying configuration files
b) Managing storage operations
c) Connecting containers to a network
d) Handling application secrets
View Answer

Answer: c
Explanation: The CNI plugin connects Pods and containers to Kubernetes networks, ensuring proper communication.

28. Which command is used to create a resource in Kubernetes?
a) kubectl get
b) kubectl create
c) kubectl apply
d) kubectl start
View Answer

Answer: b
Explanation: The kubectl create command is used to create a resource, such as pods or deployments.

29. What does the term “k8s” stand for?
a) Kubernetes clusters
b) Kubernetes orchestration
c) Kubernetes (with 8 letters removed)
d) Kubernetes container system
View Answer

Answer: c
Explanation: “K8s” is a shorthand for Kubernetes, where 8 letters are replaced by the number 8.

30. What is a Local Volume in Kubernetes?
a) A volume stored on the same node as the Pod
b) A database storage solution
c) An external network volume
d) Remote cloud storage
View Answer

Answer: a
Explanation: Local volumes in Kubernetes provide storage located directly on the node where the Pod is running, ensuring high-speed access.

31. What is the purpose of Volume Mounts in Kubernetes?
a) Connecting database clusters
b) Managing Pod metadata
c) Linking storage resources to Pods
d) Handling network routes
View Answer

Answer: c
Explanation: Volume mounts link Persistent Volumes or StorageClasses to Pods, allowing access to data storage.

32. Why do organizations prefer Canary Deployments?
a) Avoiding all network latency issues
b) Strictly random updates
c) Complete system resets
d) Faster deployment with fewer interruptions
View Answer

Answer: d
Explanation: Canary deployments enable organizations to expose changes incrementally, ensuring monitoring and quick rollback if issues arise.

33. What are Helm Charts in Kubernetes?
a) A package manager for Kubernetes applications
b) Simple storage interfaces
c) An API endpoint
d) Networking protocols
View Answer

Answer: a
Explanation: Helm Charts package Kubernetes applications and resources, making deployment and configuration management easier.

34. What is Cluster Autoscaler in Kubernetes?
a) Orchestrating network updates
b) A tool for managing internal DNS
c) Automatically adjusting the number of nodes in a cluster
d) Managing Kubernetes logs
View Answer

Answer: c
Explanation: Cluster Autoscaler dynamically scales up or down the number of nodes in a Kubernetes cluster based on demand.

35. Why is scaling important in a Kubernetes environment?
a) Restricting network access
b) Complete node shutdown
c) Optimizing resource usage and ensuring application availability
d) Faster startup times
View Answer

Answer: c
Explanation: Scaling optimizes resource efficiency and ensures that applications remain highly available and fault-tolerant.

36. What is Prometheus in Kubernetes?
a) A network routing controller
b) A scheduling component
c) A tool for service discovery
d) A monitoring system and time-series database
View Answer

Answer: d
Explanation: Prometheus is a widely used open-source monitoring system for collecting and analyzing cluster metrics.

37. What are Kubernetes Events?
a) Temporary containers
b) Inactive storage nodes
c) Logs for cluster state changes and errors
d) API service failures
View Answer

Answer: c
Explanation: Events in Kubernetes provide insights into changes, errors, and cluster state updates for monitoring.

38. What is a Kubernetes Pod Security Policy (PSP)?
a) A method for storing application logs
b) A networking configuration tool
c) A built-in scheduler
d) A deprecated feature to control security constraints for Pods
View Answer

Answer: d
Explanation: Kubernetes Pod Security Policy (PSP) was a feature that provided control over Pod-level security constraints but is now deprecated.

39. What is Kubernetes Audit Logging?
a) A scheduler for container startup
b) Logs that record all interactions with the Kubernetes API
c) A tool for data visualization
d) A tool to optimize networking interfaces
View Answer

Answer: b
Explanation: Kubernetes Audit Logs capture every interaction with the Kubernetes API server, ensuring accountability and security monitoring.

40. What is the purpose of Kubernetes CronJobs?
a) Automating scheduled tasks in a cluster
b) Deploying containers manually
c) Managing network routing
d) For scaling nodes dynamically
View Answer

Answer: a
Explanation: Kubernetes CronJobs allow you to run jobs on a scheduled basis, similar to cron jobs in Unix-like operating systems.

41. What are Kubernetes Controllers?
a) A manual scheduling service
b) A networking load balancer
c) Components that manage and automate the lifecycle of resources
d) Tools for configuring local storage
View Answer

Answer: c
Explanation: Kubernetes Controllers maintain the desired state of resources (like Deployments and ReplicaSets) by automatically managing changes.

42. What are Kubernetes Operators?
a) A scaling component
b) A logging service
c) Custom controllers that automate application management
d) Manual deployment tools
View Answer

Answer: c
Explanation: Kubernetes Operators extend the Kubernetes API to automate application lifecycle management and custom operations.

43. What are Taints and Tolerations in Kubernetes?
a) Database storage optimization
b) Control node affinity and pod placement
c) Logging interface
d) Network configuration settings
View Answer

Answer: b
Explanation: Taints and tolerations control pod placement across nodes, ensuring proper resource scheduling and availability.

44. What are Kubernetes Annotations?
a) For managing storage replicas
b) A networking configuration tool
c) A logging and debugging service
d) Key-value pairs attached to Kubernetes resources for metadata
View Answer

Answer: d
Explanation: Annotations in Kubernetes store metadata about resources that can be used for configuration and identification.

45. What is a StatefulSet in Kubernetes?
a) A type of pod
b) A way to expose services
c) A controller for managing stateful applications
d) A resource for managing network policies
View Answer

Answer: c
Explanation: A StatefulSet is a controller used for managing stateful applications that require stable, unique network identifiers and persistent storage.
