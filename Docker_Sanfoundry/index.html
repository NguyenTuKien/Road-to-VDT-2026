Docker MCQ (Multiple Choice Questions)

This set of Docker MCQs explores key topics such as containerization, Docker architecture, commands, image management, networking, and orchestration. They are useful for beginners and experienced users to prepare for interviews, certifications, and improve their skills.

1. What is Docker?
a) A Hypervisor
b) An Operating System
c) A Virtual Machine
d) A Containerization platform
View Answer

Answer: d
Explanation: Docker is a containerization platform that allows developers to package and distribute applications and their dependencies in isolated containers.

2. Which of the following is the core component of Docker?
a) Docker CLI
b) Docker Engine
c) Docker Server
d) Docker Hypervisor
View Answer

Answer: b
Explanation: The Docker Engine is the core component of Docker, which is responsible for running containers and managing Docker images.

3. Which command is used to check the version of Docker?
a) docker details
b) docker info
c) docker version
d) docker –version
View Answer

Answer: d
Explanation: The correct command to check the version of Docker installed is docker –version.

4. What is a Docker Container?
a) A network service
b) A lightweight executable package
c) An Operating System kernel
d) A Virtual Machine
View Answer

Answer: b
Explanation: A Docker container is a lightweight, standalone executable package containing everything needed to run a piece of software, including code, runtime, libraries, and system tools.

5. What does the docker ps command do?
a) Starts a Docker container
b) Displays Docker images
c) Displays Docker version
d) Displays running containers
View Answer

Answer: d
Explanation: The docker ps command lists all currently running Docker containers.

6. Which file defines the instructions to build a Docker image?
a) Dockerfile
b) requirements.json
c) config.txt
d) docker-compose.yml
View Answer

Answer: a
Explanation: The Dockerfile contains instructions for building Docker images, specifying the environment and application setup.

7. What is a Docker Image?
a) A running container
b) A snapshot of an application and its dependencies
c) A configuration file
d) A virtual hard drive
View Answer

Answer: b
Explanation: A Docker Image contains all the necessary components to run an application, including code, libraries, dependencies, and runtime.
advertisement

8. How do you create a Docker Image?
a) docker run
b) docker pull
c) docker build
d) docker start
View Answer

Answer: c
Explanation: The docker build command creates a Docker image based on the instructions provided in a Dockerfile.

9. How can you remove a Docker image?
a) docker remove
b) docker delete
c) docker rm
d) docker rmi
View Answer

Answer: d
Explanation: The docker rmi command is used to remove a Docker image from the system.

10. What is the difference between a Docker Container and a Virtual Machine?
a) Containers are slower than VMs
b) VMs run on hardware, while containers do not
c) Containers share the host OS kernel, while VMs have their own kernel
d) Containers are heavier than VMs
View Answer

Answer: c
Explanation: Containers share the host OS kernel, whereas VMs have their own OS kernel, making containers more lightweight and efficient.

11. Which command creates and starts a new container?
a) docker create
b) docker start
c) docker run
d) docker build
View Answer

Answer: c
Explanation: The docker run command creates and starts a container from a specified Docker image.

12. How do you stop a running Docker container?
a) docker pause
b) docker terminate
c) docker end
d) docker stop
View Answer

Answer: d
Explanation: The docker stop command stops a running container, gracefully terminating its processes.

13. What are the key components of Docker architecture?
a) Docker Manager, Docker Processor, Docker Configurator
b) Docker Master, Docker Node, Docker Registry
c) Docker Engine, Docker CLI, Docker Daemon
d) Docker Kernel, Docker Service, Docker Network
View Answer

Answer: c
Explanation: Docker architecture includes the Docker Engine, CLI, and Daemon, which work together to build and manage containers.

14. What is the role of the Docker Daemon?
a) It stores operating system data
b) It manages containers, images, and networks
c) It interacts with the user interface
d) It only handles logs
View Answer

Answer: b
Explanation: The Docker Daemon is responsible for managing containers, images, and network resources on the host system.

15. What is a Docker Registry?
a) A network router
b) A local storage system
c) A virtual machine host
d) A repository for storing and sharing Docker images
View Answer

Answer: d
Explanation: A Docker Registry is a storage system where you can store, share, and retrieve Docker images. Docker Hub is a popular example.

16. Which Docker service manages updates automatically?
a) Docker Daemon
b) Docker Compose
c) Docker Engine
d) Docker Plugin
View Answer

Answer: a
Explanation: The Docker Daemon manages the lifecycle and updates of containers and Docker Engine components.

17. What is a Docker Service?
a) A process that manages logs
b) A database replication tool
c) A notification system
d) A set of containers deployed together
View Answer

Answer: d
Explanation: A Docker Service is a high-level construct that deploys a set of containers together, ensuring scalability and availability.

18. What command lists all running Docker containers?
a) docker list
b) docker check
c) docker ps
d) docker stop
View Answer

Answer: c
Explanation: The docker ps command lists all currently running containers on your system.

19. Why use Multi-stage builds in Docker?
a) Increase startup time
b) Decrease image size
c) Use more memory
d) None of the above
View Answer

Answer: b
Explanation: Multi-stage builds help reduce the size of final images by removing unnecessary layers and artifacts.

20. Which CLI command shows detailed information about a container?
a) docker inspect
b) docker show
c) docker details
d) docker info
View Answer

Answer: a
Explanation: docker inspect provides detailed information about containers, including network configurations, status, and metadata.

21. What is the purpose of the docker exec command?
a) Execute a command inside a running container
b) Pull a new image
c) Remove a container
d) Start a new registry
View Answer

Answer: a
Explanation: The docker exec command allows you to run a command inside a running container, providing access to its internal environment.

22. Which of the following commands removes a stopped container?
a) docker delete
b) docker stop
c) docker prune
d) docker rm
View Answer

Answer: d
Explanation: The docker rm command removes stopped containers from your system.

23. How do you tag a Docker image with a new version?
a) docker version
b) docker update
c) docker tag
d) docker modify
View Answer

Answer: c
Explanation: The docker tag command assigns a new name and version to an existing Docker image, which is useful for version control.

24. Which of the following is true about Docker Hub?
a) It is a Docker image storage and distribution platform
b) It is a cloud service for running containers
c) It is only available for Windows
d) It is a tool for managing Docker containers
View Answer

Answer: a
Explanation: Docker Hub is a cloud-based registry service for sharing and storing Docker images.

25. What is the docker-compose tool used for?
a) Deploying single containers
b) Deleting containers
c) Running multiple containers as a service
d) Creating local network connections
View Answer

Answer: c
Explanation: docker-compose lets you define and run multi-container Docker applications with a single configuration file.

26. What is the default network mode in Docker containers?
a) Overlay
b) Static
c) Host
d) Bridge
View Answer

Answer: d
Explanation: The default network mode for a Docker container is bridge, which connects the container to a private network on the host system.

27. What is the purpose of the docker network ls command?
a) Create a new network
b) List existing networks
c) Share network bandwidth
d) Remove existing networks
View Answer

Answer: b
Explanation: The docker network ls command displays a list of all existing Docker networks on your system.

28. What is Docker Compose used for?
a) Managing multi-container applications
b) Connecting network interfaces
c) Building a single container
d) Automating system tasks
View Answer

Answer: a
Explanation: Docker Compose simplifies running multi-container applications by using a docker-compose.yml file for configuration.

29. What is the purpose of Docker Networking?
a) To deploy virtual machines
b) To increase system RAM
c) To store application data
d) To connect applications within containers
View Answer

Answer: d
Explanation: Docker networking allows containers to communicate with each other, external networks, and the host system seamlessly.

30. How do you bring up a service using Docker Compose?
a) docker-compose install
b) docker run
c) docker-compose up
d) docker-compose scale
View Answer

Answer: c
Explanation: The docker-compose up command builds and starts the containers specified in the docker-compose.yml file.

31. What is Docker Swarm?
a) A logging tool
b) A storage manager
c) An orchestration tool for managing clusters
d) A backup service
View Answer

Answer: c
Explanation: Docker Swarm is Docker’s built-in orchestration tool, enabling you to deploy, manage, and scale services across a cluster of nodes.

32. Which of the following is not a valid Docker image format?
a) iso
b) dockerfile
c) img
d) tar
View Answer

Answer: a
Explanation: Docker images are typically stored in formats such as .tar, .dockerfile, and .img, but .iso is not a valid Docker image format.

33. What is port binding in Docker?
a) Binding environment variables
b) Assigning internal ports to external ones
c) Binding memory resources
d) Binding IP addresses to containers
View Answer

Answer: b
Explanation: Port binding maps a container’s internal port to a host machine’s external port, ensuring proper communication.

34. What best practice should you follow to secure a Docker environment?
a) Use docker scan for vulnerability testing
b) Minimize container dependencies
c) Limit privileges with –privileged
d) Run containers as root
View Answer

Answer: a
Explanation: The docker scan command identifies vulnerabilities in your Docker images and helps maintain a secure environment.

35. What happens if you stop a Docker container?
a) The container is automatically destroyed
b) The container stops running but remains on the system
c) The container becomes permanently inaccessible
d) It deletes all container data
View Answer

Answer: b
Explanation: Stopping a Docker container only stops its processes, but the container remains on the system.

36. What is the purpose of a Dockerfile?
a) To create virtual machines
b) To optimize application code
c) To store environment variables
d) To define and build Docker images
View Answer

Answer: d
Explanation: A Dockerfile contains a set of instructions to define and build a Docker image, specifying dependencies, configuration, and application code.

37. Which command creates a Docker volume?
a) docker storage create
b) docker add volume
c) docker volume create
d) docker create
View Answer

Answer: c
Explanation: The docker volume create command creates a new Docker volume for persistent storage.

38. What is Docker Content Trust (DCT)?
a) A system performance monitor
b) A mechanism for local image storage
c) A debugging tool
d) A feature that verifies image authenticity
View Answer

Answer: d
Explanation: Docker Content Trust ensures that only verified and trusted images are pulled and executed, maintaining system integrity.

39. Which command is used to view logs of a running container?
a) docker logs
b) docker view
c) docker show logs
d) docker trace
View Answer

Answer: a
Explanation: The docker logs command retrieves the logs of a running or stopped container, useful for troubleshooting issues.

40. How can you interactively troubleshoot a container?
a) docker attach
b) docker exec
c) docker debug
d) docker trace
View Answer

Answer: b
Explanation: The docker exec command lets you open an interactive shell session inside a running container for real-time debugging.

41. Why is Docker commonly integrated with CI/CD pipelines?
a) Consistent environments across testing and production
b) Easier dependency management
c) Faster build and deployment times
d) All of the mentioned
View Answer

Answer: d
Explanation: Docker integration with CI/CD ensures faster deployments, dependency management, and environment consistency.

42. Why is Kubernetes often paired with Docker?
a) For debugging local machines
b) For virtualizing operating systems
c) For orchestrating large-scale container deployments
d) For building individual containers
View Answer

Answer: c
Explanation: Kubernetes orchestrates, deploys, and manages containers at scale, complementing Docker’s container creation capabilities.

43. Which tool is frequently combined with Docker for CI/CD workflows?
a) CircleCI
b) GitLab CI/CD
c) Jenkins
d) All of the mentioned
View Answer

Answer: d
Explanation: Tools like Jenkins, CircleCI, and GitLab CI/CD seamlessly integrate with Docker, streamlining CI/CD processes.

44. Which Docker command allows you to delete all unused images, containers, and networks?
a) docker remove-all
b) docker delete-all
c) docker prune
d) docker clean
View Answer

Answer: c
Explanation: The docker prune command is used to remove unused images, containers, and networks to free up space.

45. Which command starts services defined in a docker-compose.yml file?
a) docker-compose up
b) docker-compose build
c) docker-compose stop
d) docker-compose start
View Answer

Answer: a
Explanation: The docker-compose up command builds and starts containers as defined in the docker-compose.yml configuration.

46. Why should Docker images be kept small?
a) Reduced resource consumption
b) Faster startup time
c) Easier maintenance and updates
d) All of the mentioned
View Answer

Answer: d
Explanation: Smaller Docker images start faster, are easier to maintain, and consume fewer system resources.
